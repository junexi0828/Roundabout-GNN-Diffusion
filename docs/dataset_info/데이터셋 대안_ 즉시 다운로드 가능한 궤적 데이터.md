# **자율주행 상호작용 모델링을 위한 승인 불필요(Approval-Free) 궤적 데이터셋의 심층 분석 및 최적화된 획득 전략 보고서**

## **서문**

자율주행 시스템의 인지 및 예측(Perception and Prediction) 기술이 고도화됨에 따라, 단순한 물리적 충돌 방지를 넘어 복잡한 사회적 상호작용(Social Interaction)을 이해하고 예측하는 능력이 필수적인 연구 주제로 부상하였다. 특히 그래프 신경망(Graph Neural Networks, GNN)을 활용한 상호작용 모델링(Interaction Modeling) 연구는 에이전트 간의 관계를 그래프 구조로 표현함으로써 비정형적인 환경에서의 주행 의도를 파악하는 데 탁월한 성능을 보이고 있다. 그러나 이러한 연구의 가장 큰 병목 구간은 아이러니하게도 알고리즘의 설계가 아닌, 양질의 데이터 확보에 있다. Waymo Open Dataset, Argoverse 2, INTERACTION 데이터셋 등 최신 대규모 데이터셋들은 정교한 센서 데이터를 제공하지만, 기업의 엄격한 라이선스 동의 절차, 승인 대기 시간, 혹은 상업적/비상업적 용도 제한 등으로 인해 연구의 초기 진입 장벽을 높이는 요인이 되고 있다.

본 보고서는 이러한 배경 하에, 별도의 승인 절차 없이 즉시 접근 가능한(Approval-Free) 오픈소스 데이터셋들을 면밀히 분석하고, 특히 회전교차로(Roundabout)와 같은 고난이도 위상(Topology)을 가진 데이터를 확보하기 위한 최적의 전략을 제시하는 것을 목표로 한다. 이를 위해 Stanford Drone Dataset(SDD)의 'Death Circle', NGSIM의 'Lankershim Boulevard', 그리고 주요 GNN 모델(Social-STGCNN, Trajectron++)의 GitHub 리포지토리에 내장된 전처리 데이터셋을 심층 조사하였다. 본 분석을 통해 도출된 데이터 획득 전략은 불필요한 행정적 절차를 제거하고, 연구자가 즉시 씬 그래프(Scene Graph) 구축 및 모델 학습 단계로 진입할 수 있도록 하는 구체적인 로드맵을 제공한다.

## ---

**1\. Stanford Drone Dataset (SDD) 심층 분석: 'Death Circle' 위상과 데이터 가용성**

Stanford Drone Dataset(SDD)은 조감도(Bird's Eye View) 시점에서 촬영된 대규모 궤적 데이터셋으로, 차량 중심의 기존 데이터셋과 달리 보행자, 자전거, 스케이트보더, 카트 등 다양한 이기종 에이전트(Heterogeneous Agents)가 혼재된 캠퍼스 환경을 담고 있다는 점에서 독보적인 가치를 지닌다. 특히 본 연구의 핵심 관심사인 'Death Circle' 씬은 비신호 회전교차로에서의 복잡한 협상(Negotiation) 과정을 포함하고 있어 GNN 기반 상호작용 모델링의 최적의 벤치마크로 평가된다.

### **1.1 데이터 접근성 분석: 공식 경로와 GitHub 미러링**

과거 SDD는 스탠포드 대학교의 CVGL(Computational Vision and Geometry Lab) 공식 웹사이트를 통해 배포되었으나, 대용량 비디오 파일의 호스팅 문제와 링크 소실(Link Rot) 현상으로 인해 연구자들의 접근이 제한되는 경우가 빈번하였다. 본 조사를 통해, 연구 커뮤니티에 의해 구축된 'GitHub 미러(Mirror)' 리포지토리들이 데이터 획득의 가장 신속하고 안정적인 경로임이 확인되었다.

#### **1.1.1 GitHub을 통한 승인 불필요(Approval-Free) 접근 매커니즘**

flclain/StanfordDroneDataset 1 및 Open\_Datasets 관련 포크 리포지토리들은 SDD의 핵심인 궤적 어노테이션(Annotation) 텍스트 파일들을 압축 해제된 상태로 직접 호스팅하고 있다. 이는 연구자가 별도의 가입 절차, 승인 요청, 혹은 대용량 압축 파일 다운로드 후의 해제 과정 없이 git clone 명령어 하나만으로 필요한 씬(Scene)의 데이터를 즉시 확보할 수 있음을 의미한다.

* **접근성 평가:** 최상 (Immediate Availability)  
* **승인 요구사항:** 없음 (공개 리포지토리)  
* **라이선스:** CC-BY-NC-SA 3.0 (저작자 표시-비영리-동일조건변경허락). 학술적 연구 목적의 자유로운 사용과 수정이 허가됨.2  
* **데이터 지속성:** 대학 서버의 링크가 만료되더라도 GitHub의 분산 버전 관리 시스템 특성상 데이터의 영구적인 접근성이 보장된다. 이는 연구의 재현성(Reproducibility) 확보 측면에서 매우 중요한 이점이다.

### **1.2 'Death Circle' 씬(Scene)의 위상학적 특징과 GNN 연구 적합성**

'Death Circle'은 스탠포드 캠퍼스 내에 위치한 악명 높은 회전교차로(Roundabout) 구역이다. 'Nexus'나 'Bookstore'와 같은 다른 SDD 씬들이 주로 직선형 도로나 광장 형태를 띠는 것과 달리, Death Circle은 명확한 원형 위상을 가지며 에이전트 간의 끊임없는 진입-진출 협상을 강제한다.

* **위상(Topology):** 비신호 회전교차로 (Unsignalized Roundabout). 신호등에 의해 통제되는 이산적(Discrete) 상호작용이 아닌, 에이전트 간의 눈치 싸움과 양보(Yielding)가 연속적으로 발생하는 환경이다.  
* **에이전트 구성:** 보행자, 자전거 이용자, 차량, 스케이트보더, 골프 카트 등이 뒤섞인 고밀도 이기종 환경이다. 이는 GNN이 단순히 거리 기반의 엣지(Edge)를 생성하는 것을 넘어, 에이전트의 유형(Node Type)에 따른 상호작용 가중치를 학습해야 함을 시사한다 (예: 차량은 보행자에게 양보하지만, 자전거와는 경쟁할 수 있음).3  
* **연구 가치:** 회전교차로에서의 궤적 예측은 원심력과 구심력이 작용하는 물리적 운동 모델과 사회적 규칙이 결합된 복합적인 문제다. SDD의 Death Circle 데이터는 이러한 물리-사회적 결합 모델(Physics-Social Coupled Model)을 검증할 수 있는 거의 유일한 '승인 불필요' 데이터셋이다.

### **1.3 궤적 데이터 포맷 상세 분석**

GitHub 리포지토리를 통해 확보 가능한 데이터는 annotations.txt 파일 형식을 따르며, 이는 별도의 파싱 라이브러리 없이도 판다스(Pandas)나 넘파이(NumPy)로 즉시 로드 가능한 공백 구분(Space-Separated) 텍스트 포맷이다.1

#### **1.3.1 스키마(Schema) 정의 및 기술적 해석**

각 행(Line)은 특정 프레임에서의 단일 객체 상태를 나타내며, 다음과 같은 컬럼 구조를 가진다.

| 인덱스 | 필드명 | 데이터 타입 | 설명 및 GNN 활용 방안 |
| :---- | :---- | :---- | :---- |
| **0** | Track ID | Integer | 궤적 식별자. 동일한 ID를 가진 행들을 그룹화하여 시계열 텐서 $T\_{obs}$를 구성하는 기준 키(Key)로 사용된다. |
| **1** | xmin | Integer | 바운딩 박스 좌상단 X 좌표 (픽셀 단위). |
| **2** | ymin | Integer | 바운딩 박스 좌상단 Y 좌표 (픽셀 단위). |
| **3** | xmax | Integer | 바운딩 박스 우하단 X 좌표 (픽셀 단위). |
| **4** | ymax | Integer | 바운딩 박스 우하단 Y 좌표 (픽셀 단위). |
| **5** | frame | Integer | 비디오 프레임 번호. 시간 $t$를 나타내며, GNN의 시간적 엣지(Temporal Edge)를 구성하는 데 필수적이다. |
| **6** | lost | Binary (0/1) | 객체가 시야 밖으로 사라졌거나 가려져 추적을 놓친 상태. GNN 학습 시 이 값이 1인 프레임은 마스킹(Masking) 처리하여 노이즈 학습을 방지해야 한다. |
| **7** | occluded | Binary (0/1) | 다른 객체나 지형지물에 의해 가려진 상태. 강건한 모델링을 위해 오클루전 상황에서의 궤적 보간(Interpolation) 성능 평가 지표로 활용될 수 있다. |
| **8** | generated | Binary (0/1) | 시스템에 의해 자동 보간된 데이터 여부. Ground Truth로서의 신뢰도가 다소 낮을 수 있음을 의미한다. |
| **9** | label | String | 객체 클래스 (예: "Biker", "Pedestrian", "Car"). 이기종 그래프(Heterogeneous Graph) 구축 시 노드 속성(Node Attribute)으로 임베딩된다. |

#### **1.3.2 전처리 요구사항: 중심점 변환 및 좌표계 투영**

GNN 모델은 일반적으로 바운딩 박스가 아닌 객체의 중심점(Centroid) 궤적을 입력으로 받는다. 따라서 annotations.txt 로드 후 다음과 같은 변환 과정이 필수적이다.

1. 중심점 계산:

   $$C\_x \= \\frac{x\_{min} \+ x\_{max}}{2}, \\quad C\_y \= \\frac{y\_{min} \+ y\_{max}}{2}$$  
2. **픽셀-미터 변환 (Homography):** SDD는 드론에서 촬영된 영상 좌표계(Pixel)를 사용하므로, 실제 물리적 거리(미터) 기반의 상호작용을 모델링하기 위해서는 호모그래피 행렬(Homography Matrix, $H$)을 이용한 투영 변환이 필요하다. Trajectron++ 등의 리포지토리에는 각 씬에 대한 $H$ 행렬 파일(H.txt)과 변환 로직이 포함되어 있어 이를 활용해야 한다.5

**2\. NGSIM 'Lankershim Boulevard': 도심 교차로 데이터의 재조명**

NGSIM(Next Generation Simulation) 데이터셋은 교통 공학 분야의 고전적인 데이터셋으로, 미 연방도로청(FHWA) 주도하에 수집되었다. 사용자 질의에서 언급된 'Lankershim Boulevard' 데이터에 대한 접근성과 GNN 활용 사례를 분석한 결과, 일반적인 오해와 달리 회전교차로가 아닌 신호 교차로(Signalized Intersection)임이 확인되었다.

### **2.1 데이터 접근성: 미 교통부(USDOT)를 통한 완전 개방**

NGSIM 데이터셋은 미국 정부의 오픈 데이터 정책에 따라 **USDOT ITS DataHub**(data.transportation.gov)를 통해 대중에 완전히 공개되어 있다.7

* **접근 방식:** 별도의 연구 계획서 제출이나 승인 대기 없이 웹사이트에서 'Export' 버튼을 눌러 CSV 형식으로 즉시 다운로드 가능하다.9 또한 Socrata Open Data API(OData)를 통해 프로그래밍 방식으로 데이터를 쿼리할 수도 있다.  
* **포맷:** 궤적 데이터는 .csv 포맷으로 제공되며, 각 행은 0.1초 단위의 차량 상태를 나타낸다. 주요 컬럼으로는 Vehicle\_ID, Frame\_ID, Local\_X, Local\_Y, Global\_X, Global\_Y, v\_Length, v\_Width, v\_Class, v\_Vel, v\_Acc 등이 포함된다.8

### **2.2 위상학적 정정: 회전교차로가 아닌 신호형 간선도로**

많은 연구자들이 복잡한 상호작용 데이터셋을 검색하는 과정에서 NGSIM Lankershim을 회전교차로 데이터와 혼동하거나 그룹화하는 경향이 있다. 그러나 문헌 및 메타데이터 분석 결과, Lankershim Boulevard 데이터셋은 \*\*로스앤젤레스의 유니버설 시티(Universal City) 구역 내 약 500미터 구간의 간선도로(Arterial Road)\*\*를 대상으로 한다.10

* **도로 구조:** 편도 3-4차로의 도심 간선도로이며, Universal Hollywood Drive 등을 포함한 4개의 \*\*신호 교차로(Signalized Intersections)\*\*를 포함한다.11  
* **상호작용 특성:** 회전교차로의 '양보(Yielding)' 및 '갭 수락(Gap Acceptance)'과 같은 비신호 상호작용보다는, 신호등에 의한 정지-출발(Stop-and-Go), 대기열 형성(Queueing), 그리고 차로 변경(Lane Changing) 행태가 주를 이룬다.  
* **결론:** 만약 연구의 목적이 순수하게 '회전교차로 내에서의 상호작용'을 모델링하는 것이라면, Lankershim 데이터셋은 위상학적으로 부적합하다. 그러나 도심 내에서의 **차량 추종(Car-Following)** 및 **끼어들기(Cut-in)** 예측 모델링에는 매우 유용한 데이터다.

### **2.3 GNN 연구 활용 사례 및 노이즈 이슈**

Lankershim 데이터는 회전교차로가 아님에도 불구하고 GNN 기반 상호작용 모델링 연구에서 빈번하게 활용되고 있다. 이는 도심 환경의 혼잡 상황(Congested Traffic)을 잘 반영하고 있기 때문이다.

* **활용 사례 (Interaction Modeling):**  
  * **차로 변경 예측:** 주변 차량(이웃 노드)의 속도와 위치 정보를 엣지 속성(Edge Attribute)으로 하여 대상 차량의 차로 변경 의도를 예측하는 GNN 모델 학습에 사용된다.10  
  * **궤적 예측:** 차량을 노드로, 상호작용을 엣지로 정의한 그래프 구조에서 GAT(Graph Attention Network) 등을 적용하여 미래 위치를 예측하는 벤치마크로 활용된다.  
* **데이터 노이즈 및 전처리:** NGSIM 데이터는 2005년에 수집된 영상 기반 데이터로, 궤적 추출 과정에서의 재구축 노이즈(Reconstruction Noise)가 심한 것으로 알려져 있다. 따라서 현대적인 GNN 연구에서는 원본 데이터 대신 I-80 및 US-101 데이터를 평활화(Smoothing) 처리한 버전이나, 이를 대체하기 위해 나온 **HighD**(드론 데이터) 등을 선호하는 추세다.12 하지만 HighD는 고속도로 데이터이므로, 도심 환경 연구에서는 여전히 Lankershim이나 SDD가 선호된다.

**3\. GitHub 리포지토리 내 'Pre-processed' 데이터셋 조사: 연구 가속화의 열쇠**

원시 데이터(Raw Data)를 다운로드하여 파싱하고 정제하는 과정(ETL)은 연구 초기 단계에서 막대한 시간을 소모하게 한다. 본 조사를 통해 확인된 가장 효율적인 대안은, 주요 SOTA(State-of-the-Art) 모델의 GitHub 리포지토리에 포함된 **전처리된(Pickled/Pre-processed)** 데이터셋을 활용하는 것이다.

### **3.1 Social-STGCNN 리포지토리: 즉시 학습 가능한 텐서 데이터**

abduallahmohamed/Social-STGCNN 리포지토리는 GNN 기반 궤적 예측의 기준점이 되는 연구로, 해당 리포지토리 내 datasets/ 폴더 구조는 데이터 획득의 보고(Treasure Trove)다.13

* **포함 데이터:** ETH 및 UCY 데이터셋이 기본적으로 포함되어 있으며, 이는 텍스트 파일 형태 혹은 넘파이(NumPy) 텐서로 전처리되어 있다.  
* **SDD 데이터 처리:** SDD의 경우 파일 크기로 인해 원본 전체가 포함되어 있지는 않으나, utils.py 스크립트 내에 StanfordDroneDataset 클래스가 구현되어 있어, annotations.txt 파일만 datasets/ 폴더에 위치시키면 자동으로 그래프 텐서로 변환해주는 로더(Loader)가 준비되어 있다.14  
* **활용 전략:** 이 리포지토리를 클론(Clone)함으로써, 복잡한 좌표 변환 로직을 직접 구현할 필요 없이 utils.py의 함수를 호출하여 데이터를 로드할 수 있다.

### **3.2 Trajectron++ 리포지토리: 이기종 데이터 처리의 표준**

StanfordASL/Trajectron-plus-plus 리포지토리는 SDD와 NuScenes 데이터를 처리하는 데 있어 가장 강력하고 포괄적인 전처리 파이프라인을 제공한다.15

* **핵심 스크립트 (process\_data.py):** 이 스크립트는 원시 텍스트 데이터를 읽어들여 판다스 데이터프레임으로 변환하고, 이를 다시 피클(.pkl) 파일로 직렬화(Serialization)하는 기능을 수행한다.5  
  * **기능 1:** 호모그래피 행렬을 적용한 미터 단위 변환 자동화.  
  * **기능 2:** 씬(Scene)별로 훈련/검증/테스트 세트 분할 자동화.  
  * **기능 3:** NuScenes 데이터셋의 맵 정보(Map API)와 연동하여 도로 구획 정보를 그래프 노드 속성으로 추가하는 기능 포함.16  
* **데이터 위치:** experiments/pedestrians/data 경로에 ETH/UCY 및 SDD 처리를 위한 스켈레톤 코드가 위치하며, 사용자는 원시 데이터를 raw/ 폴더에 넣고 스크립트만 실행하면 된다.  
* **의의:** 이 리포지토리는 단순한 데이터 로더를 넘어, **씬 그래프(Scene Graph) 생성기**로서의 역할을 수행한다. 이기종 에이전트(차량 vs 보행자)의 타입을 구분하여 노드 속성에 저장하므로, 이기종 그래프(Heterogeneous Graph) 연구에 최적화되어 있다.

### **3.3 EigenTrajectory 및 SGCN 리포지토리**

InhwanBae/EigenTrajectory 리포지토리는 README를 통해 "Preprocessed ETH and UCY datasets are included in this repository"라고 명시하고 있어, 별도의 다운로드 없이 즉시 데이터를 확보할 수 있는 가장 빠른 경로 중 하나다.17 shuaishiliu/SGCN 또한 dataset/ 폴더에 처리된 데이터를 포함하고 있다.18

### **4.2 최종 선정: Stanford Drone Dataset (SDD)**

**결론적으로, SDD의 'Death Circle'이 INTERACTION/Waymo를 대체할 최적의 '무승인' 데이터셋으로 선정되었다.**

* **선정 이유 1 (접근성):** GitHub 미러를 통해 git clone만으로 획득 가능하므로 행정적 소요 시간이 '0'이다.  
* **선정 이유 2 (위상 적합성):** 사용자가 요구한 '회전교차로' 데이터를 포함하는 유일한 무승인 데이터셋이다. rounD 데이터셋이 품질 면에서는 더 우수할 수 있으나(드론 전용), 신청 절차가 필요하다는 점에서 '무승인' 요건에 미달한다.  
* **선정 이유 3 (상호작용의 풍부함):** Death Circle은 차량과 자전거, 보행자가 섞여 있어 단일 차종 위주의 NGSIM보다 훨씬 복잡하고 흥미로운 엣지(Interaction Edge)를 형성한다.

## ---

**5\. 수정된 연구 계획: 데이터 획득에서 씬 그래프 변환까지**

선정된 SDD 'Death Circle' 데이터셋과 Trajectron++ 리포지토리의 전처리 도구를 결합하여, 승인 대기 시간 없이 즉시 연구를 시작할 수 있는 구체적인 실행 계획을 수립한다.

### **Phase 1: 하이브리드 데이터 획득 (Zero-Wait Acquisition)**

전통적인 '웹사이트 다운로드' 방식을 탈피하고, GitHub를 '데이터 CDN'처럼 활용한다.

1. **Trajectron++ 리포지토리 클론:**  
   Bash  
   git clone \--recurse-submodules https://github.com/StanfordASL/Trajectron-plus-plus.git  
   cd Trajectron-plus-plus

   이 단계에서 데이터 처리 로직(process\_data.py)과 모델 구조를 확보한다.  
2. SDD 어노테이션 데이터 페칭 (Fetching):  
   대용량 비디오 파일 대신, flclain/StanfordDroneDataset 리포지토리에서 텍스트 어노테이션만 선별적으로 다운로드한다.  
   Bash  
   git clone https://github.com/flclain/StanfordDroneDataset.git temp\_sdd  
   mkdir \-p experiments/pedestrians/raw/sdd/deathCircle  
   cp \-r temp\_sdd/deathCircle/video\* experiments/pedestrians/raw/sdd/deathCircle/

   이로써 수 기가바이트의 영상을 다운로드할 필요 없이 수 메가바이트의 텍스트 파일만으로 학습 준비를 마친다.

### **Phase 2: 미터 단위 씬 그래프 변환 (Metric Scene Graph Transformation)**

원시 픽셀 데이터를 GNN이 이해할 수 있는 물리적 상호작용 그래프로 변환한다.

1. 호모그래피 변환 실행:  
   Trajectron++ 내장 스크립트를 실행하여 픽셀 좌표 $(u, v)$를 호모그래피 행렬 $H$에 투영, 미터 좌표 $(x, y)$로 변환한다.  
   Bash  
   python experiments/pedestrians/process\_data.py \--data sdd \--output\_path data/processed

   이 과정에서 'Death Circle'의 물리적 크기가 복원되어, 에이전트 간의 실제 거리 기반 상호작용 계산이 가능해진다.  
2. 씬 그래프($G\_t$) 구축:  
   변환된 데이터를 기반으로 각 타임스텝 $t$마다 그래프 $G\_t \= (V\_t, E\_t)$를 정의한다.  
   * **노드 ($V\_t$):** 현장에 존재하는 모든 에이전트. 속성 벡터 $h\_i \=$.  
   * **엣지 ($E\_t$):** 상호작용 연결. Death Circle의 특성을 반영하여, 단순 거리($\<10m$) 기반 연결뿐만 아니라, \*\*'시야각(Field of View)' 기반 유향 엣지(Directed Edge)\*\*를 생성한다. (예: 진입 차량은 회전 중인 차량을 볼 수 있으나, 회전 중인 차량은 진입 차량을 보지 못할 수 있음).

### **Phase 3: GNN 모델 학습 및 검증**

1. **데이터 분할:** Death Circle의 video0\~video3을 학습용으로, video4를 테스트용으로 사용하여 일반화 성능을 검증한다.  
2. **모델링:** Spatio-Temporal GCNN (ST-GCNN) 구조를 채택하되, 회전교차로의 특성을 반영하여 \*\*중심점(Center Point)\*\*을 가상의 글로벌 노드로 추가, 모든 에이전트가 중심점과의 관계(원심력/구심력 방향)를 학습하도록 유도한다.  
3. **평가 지표:** ADE(Average Displacement Error)와 FDE(Final Displacement Error) 외에, 회전교차로 특화 지표인 \*\*'진입 시 양보 준수율(Yield Compliance Rate)'\*\*을 추가적으로 분석한다.

## **결론**

본 연구 보고서는 자율주행 연구의 핵심 자원인 데이터셋 확보 과정에서 발생하는 행정적 장벽을 우회하고, 연구의 본질인 모델링에 즉시 착수할 수 있는 실질적인 방안을 제시하였다. 분석 결과, \*\*SDD의 'Death Circle'\*\*이 회전교차로 상호작용 연구를 위한 최적의 '승인 불필요' 데이터셋임이 입증되었으며, **Trajectron++ 리포지토리**를 활용한 하이브리드 데이터 획득 전략이 가장 효율적인 파이프라인으로 식별되었다. NGSIM Lankershim 데이터는 회전교차로 연구에는 부적합하나 도심 교통류 연구에는 여전히 유효하다는 점 또한 명확히 하였다. 제안된 연구 계획을 통해 연구자는 데이터 승인 대기라는 비생산적인 시간을 제거하고, 즉각적으로 씬 그래프 구축과 GNN 모델 고도화 단계로 진입할 수 있을 것이다.
